义务维修

前言

  • 硬件检测与基本的维修，主要是针对台式。若对方笔记本在保修期内，直接售后。
  • 基础知识不涉及电焊、电路板排错，出现此问题建议客户直接更换部件，或另找维修
  • 系统的安装，常见的软件安装问题，系统环境、驱动配置，以及系统的基本操作。
    • 虚拟机安装系统，以及物理机安装系统
    • 老主板安装系统
  • 如果有空可以讲讲办公室经常出现的网络排错

学习预告

  • 学习只顾及维修硬件部分知识，只要求掌握其基本的作用以及容易出现问题的情况

  • 检查硬件错误：一般此类硬件错误可以不更换核心部件

  • 提出分析报告和维修建议：预计是哪里出现问题，将测试情况以及得出的结论，便于后续客户去买进设备和给下一个维护者便利

  • 安装系统有几种常见的方法：在不同的时候选择最适合的方法

  • 系统分区建议

  • 后续操作系统的一些常见问题：驱动安装，上不了网，基本信息排错

  • 网络排错

    ==使用百度解决问题==

硬件部分

硬件维修的目的：进入BIOS系统，因为能进入BIOS后，剩下的都好排错和重装

常见的硬件基本功能(了解)

  • CPU：- 对数据进行运算处理
  • 主板： - 连接各个硬件的平台，提供I/O接口
  • 内存： - 存放CPU运算数据及硬盘与外部数据交换
  • 硬盘： - 提供数据存储
  • 显卡： - 可选 ： 提供图形数据处理 & 深度学习训练 & 打游戏
  • 电源： - 交流转直流为其他组件提供供电
  • 散热器： - CPU GPU等高发热量器件提供散热
  • 外设 ： - 键盘鼠标，等外置设备
  • 显示屏 ： - 提供系统可视化界面

在以上常见的设备中，容易出现故障的有前：CPU,主板，内存，硬盘

BIOS

BIOS 它保存着系统CPU、软硬盘驱动器、显示器、键盘等部件的信息。 关机后，系统通过一块后备电池向CMOS供电以保持其中的信息

  • BIOS：保存着日期信息，如果每次重启发现时间重置，可以更换电池
  • 开机时根据不同主板类型按快捷键进入，常见快捷键有F2，F1，del，F12，机型百度：联想BIOS快捷键
  • 学习目标：能够进入BIOS进行配置，能够对常见BIOS问题进行解决

  • BIOS：（基本输入输出系统）的学习

    • 可以理解为最基础的配置驱动
    • 它保存着系统CPU、软硬盘驱动器、显示器、日期和时间的信息。 关机后，系统通过一块后备电池向CMOS供电以保持其中的信息

  • 常见的配置:

    • 选择硬盘启动项：系统从哪个盘启动，进入PE的方法
    • 选择显示输出方式：从显卡输出还是集显、放电后解决显示器不输出（不带集显的除外）、
    • 选择开启SATA硬盘AHCI模式：部分老版笔记本进入PE后看不见硬盘，需要进入Devices->ATA Drives Setup -> SATA Mode 设为AHCI mode
    • 开启英特尔虚拟化：给同学装虚拟机遇到最多的

  • 重置BIOS：F9，保存：F10

显示输出方式（了解/需要时百度）

  • 关键词：Graphic Device，Graphic Adapter。意思：显示驱动。

  • 选项解释：

    • on-board graphic：启用禁用内置显卡（若CPU支持）
    • IGD：内置集成显卡
    • PED：PCIE显卡
    • PCI/IGD：优先PCI显卡，若无则用集成显卡
    • PCI/PED：优先PCI显卡，若无则用独立PCIE显卡

显示器连接线

学习目的：了解外形张啥样。（其他传输模式和速率有兴趣可以自行百度）

  • 显示器连接线常见为3种：HDMI、VGA、DVI线
  • 其他知识点：
    • HDMI：适合影像传输的专用型数字化接口。能够传输视频和音频

英特尔虚拟化

学习目的：开不了虚拟机。。

这里新版的BIOS容易出错，设置时，应该进入高级选项进行配置

常见关键词：虚拟化、VT、VT-x、VT-D、Inter(R) Virtualizantion Technology

找不到就百度：xxx笔记本/主板 开启虚拟化技术

排错的技巧

  • 排错：目的是为了找到出现问题的地方

  • 常见的问题有：开不了机，开机后无限重启，开机后不显示

  • 学习目标：逐步排错，找到出现问题的硬件和可能的问题，配合BIOS进行，什么是放电

  • 介于本门课是实操大于理论，这里给大家过一遍解决问题的方法
  • 学习建议：讲过一边知道这项错误，我脑子里有印象，讲过这个问题，这样就能快速定位问题

接着，举例几个最常见的例子，还有排错流程，此流程可以解决绝大多数非硬件损坏问题

大概率出现的问题

“开机后无限重启”，“开机后不显示屏幕”

排错方法

  • 此类问题大部分情况是由于内存松动产生的

    • 拔下内存条，使用橡皮擦擦拭金手指
    • 将内存条卡卡扣插入到位

重启计算机，若还是反复重启，则需要对主板进行放电操作

  • 一般硬件更新导致的系统无法开机，都使用对主板放电进行修复

    注意：必须要在断电的情况下进行操作，断电是指拔掉电源线，不是把插排上的开关关了

    • 找到主板上的电池，扣下（主板上的电池有啥用）

      • 情况一：主板上有CMOS_REST 的两个金属触（一般出现在主板拐角处，单独的两个触），不同主板叫法不一样，意思是是（BIOS重置），此时直接用导电螺丝刀触碰两个金属针脚进行放电。可能出现的关键词有：RESET（重置），BIOS，CMOS,CLR

      • 情况二：没有找到金属针脚，使用螺丝刀等导电体连接电池槽正负极，进行放电操作

      • 情况三：有一个塑料的跳线帽,取下后短接（螺丝刀触碰）放电

    • 放电时：长按电源开关机键，一般重复2-3次

    • 开启后若成功解决“无限重启”。将电池放入，完成硬件维修。

    • 再次开机，将会自动进入BIOS页面，查看右下角，选择使用默认配置F9：default setting，再F10： 保存并重启

衍生问题

经过放电操作后可以开机，但不显示屏幕

  • 此类问题可能是显卡在BIOS配置不正确导致的
    • 检测方法：使用键盘插到主板后侧USB插口，开机瞬间是否三盏灯亮起，若亮起则定位为此问题
    • 更换显示器口的位置，从显卡插口处弄到主板插口处
  • 若能成功进入系统则确定位置为显卡问题
    • 出现该问题也不需要慌，一般显卡拆下来放一会，再将BIOS内输出方式调回显卡即可

其他排错法

  • 排列组合排错：两条内存。一条条插在不同插槽、或其他能用的电脑。
    • 目的：排除插槽、内存条问题
  • 排除法：只有内存和CPU，再接显示器、键盘。就可以显示BIOS页面，不需要硬盘、显卡等！
    • 目的：排除硬盘、显卡故障，再通过BIOS进行检查
  • 组合法：键盘可以查看硬盘/系统 是否被正常启动，启动时表现为：三个灯瞬间亮起后灭掉，再亮起第一盏灯
    • 目的：通过这个可以排除硬盘问题，进一步从硬盘排错到显卡排错

网络排错

学习目标：

  • 了解常见的网络错误
  • 网络出现的问题
  • 网络排错方法
  • 防火墙开关和造成的影响
  • 简单的局域网排错

网络名词（知识点）

  • IP 最常见的地址
    • 局域网知识：内网IP与子网掩码
    • 学习目标：如何设置IP，什么是自动分配，常见的排错命令
  • 网关 所有数据都需要经过它
    • 一般是路由器地址
    • 学习目标：如何找到网关
  • DNS 配置：解析域名，上不了网的一种情况
    • 测试：ping www.baidu.com
  • DHCP ：分配IP地址
  • 防火墙与网络发现：防火墙配置不当或不配置会导致很多服务不能用，导致排错时出现误判

内网地址

  • 常见的地址192.168.1.1,192.168.3.1（应用于简单的局域网）
    • 一般网关地址结尾是 1 或 254 如当前设备的IP为192.168.1.5 ，可能的网关为192.168.1.1 或 192.168.1.254
  • 通过ipconfig 查看IP，通过ping检测连通性，关闭防火墙
  • 实验:找到自己的内网IP，找到自己网关地址，登陆网关页面
    • 安全问题：常见问题，用户：admin 密码123456、654321

物理设备

  • 无线路由器：分为WAN口和LAN口
    • WAN 蓝色 接调制解调器（猫，Modem）
    • LAN 黄色 接设备的
    • （现在高端设备每个口都可以支持WAN和LAN，还支持自动识别）
  • 无线路由器重置：用针长按路由器REST口几秒后路由器灯闪烁
    • 查看背板的WIFI名和默认密码，通过浏览器访问路由器（网关）IP

情景展示

学习前提：知道什么是DHCP

DHCP：IP地址动态分配，指不需要手动配置 IP 的情况下能够自动获取到 IP 地址

在家庭网络中：

  • 一般需要使用WAN口接猫，然后使用拨号账号进行上网
  • 或者接入WAN口使用DHCP(自动分配地址)进行分配

在办公室网络中：(需要了解原理)

  • 外部网络接入LAN口后对网络进行桥接
  • 外部网络接入WAN口后再对内网进行分配
  • 外部网络接入WAN口配置WAN口的IP，然后对内网进行分配
  • （理解为我们中继，画图理解，不理解这里后面很难排错）

学习目标：

  • 了解办公室家庭的拓扑结构
  • 了解路由器的两个接口作用

排错实例过程

注意：这里只讲一个基础的过程，现实排错中应该需要组合下述方法

故障排除

  • 确定网线是否可用：
    • 使用路由器排错：直接将网线两端插入路由器背板，查看两个端口，灯是否亮起
    • 操作时可以拔掉其他的的网线避免短暂的广播风暴
  • 连通性检查：
    • 通过ping命令检测连通性
    • 检测自身和网关的连通性
    • 检测自身和局域网设备的连通性
    • 易错点：开了防火墙防火墙，和关闭了网络发现

路由器问题：

  • 路由器是否能分到地址，配置是否错误
  • 通过电脑直接连接入户网线，来排除是路由器问题还是入户口问题

DNS问题：

  • 能ping通网关和外网网址
  • 无法ping通域名

故障修复

  • 重置路由器，重新给他们配置
