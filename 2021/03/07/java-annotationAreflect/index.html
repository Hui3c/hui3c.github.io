注解

注解的定义

注解(Annontation):

  • 从jdk5.0引入的新技术
  • 注解可以被编译器读取
  • 注解有检查的约束
// 重写注解
@Override
public String toString() {

}

定义一个注解

元注解

  • 通过反射机制实现对元数据的访问呢
  • 元注解有四个：用于自定义注解，比如我需要创建一个自己的注解
  • (使用默认ctrl点进去，再点里面的值可以看见所有选项)
  • @Target ：注解可用于哪里（如类还是接口还是对象）
  • @Retention：在什么界别保存信息，描述生命周期，一般开发自定义使用RUNTIME，运行时
  • @Documented：将注解包含到javadoc中
  • @Inherited： 可以继承父类中的该注解

@Target:表示注解可以用在哪些地方

// 以数组传参，可以用在类和方法上
public class Test {
    // 一个类中只能有一个pubic
}
// 定义一个自定义的注解
@Target({ElementType.TYPE,ElementType.METHOD})
@interface MyAnnontation {

}

@Retention：运行界别，表示注解在什么地方还有效

  • 有源码时有效SOURCE，类创建时有效CLASS，运行时有效RUNTIME
  • 自定义的类默认写RUNTIME
  • RUANTIME > CLASS > SOURCE：从底层源码到编译，到目标程序
@MyAnnontation(school = {"123"})
public class Test {
}
 // --------
// 定义作用域
@Target({ElementType.TYPE,ElementType.METHOD})
@Retention(RetentionPolicy.RUNTIME)
@interface MyAnnontation {

    // 注解参数： 参数类型 + 参数名()，default 为默认值
    String name() default "";
    int age() default 0;
    String[] school();
}

隐藏属性：当只有一个参数时可以在定义时使用value名，这样在赋值时就不需要指定名字

// 注解参数
String value() default "";

// 传参方式
@MyAnnontation("字符串")
// 等价
@MyAnnontation(value = "字符串")

反射 Reflection

静态语言与动态语言

动态语言：

运行时可以改变结构的语言，运行时代吗可以根据某些条件改变自身结构

主要的动态语言有：Object-C、C#、JavaScript、PHP、Python

// javascript 是动态语言
function() {
    // 定义时是字符串
    var str = "var a = 1; var b = 2; alert(a+b);"
    
    // 执行完后后str的值就变成了3
    eval(str)
}

静态语言：

运行时结构不可变。如Java、C、C++

Java 不是动态语言，称为“准动态语言”。通过反射机制获得类似动态语言的特性

反射的特点

  • 允许程序执行期间借助Reflection API取得任何类的内部信息
  • 缺陷：损失一些安全性。是一种解释操作对性能有影响，

反射提供的功能

  • 运行时判断任意对象所属类
  • 运行时构造任意一个类的对象
  • 运行时判断任意一个类所具有的成员变量和方法
  • 运行时处理注解
  • 生成动态代理
  • …

一个类在内存中只有一个class对象，类被加载后，类的整个结构都会被封装在class对象中

public class Test2 {
    public static void main(String[] args) throws ClassNotFoundException {
        // 通过反射获取类的class 对象
        Class c1 = Class.forName("com.hui.pojo.User");
        Class c2 = Class.forName("com.hui.pojo.User");
        Class c3 = Class.forName("com.hui.pojo.User");

        // 一个类在内存中只有一个class对象
        System.out.println(c1.hashCode());
        System.out.println(c2.hashCode());
        System.out.println(c3.hashCode());
    }
}

Class 类

  • 只能通过系统建立，本身也是一个类
  • 加载类在JVM中指挥有Class实例
  • 通过Class 可以完整地得到一个类中的所有被加载的结构
  • Class 类是Reflection 的根源，针对任何动态加载、运行的类，唯有先获得相应的Class对象

通过反射获取类

package com.hui.pojo;

public class Test03 {
    public static void main(String[] args) throws ClassNotFoundException {
        // 实例化
        Person person = new Stuent();

        // 方法一：通过对象获得
        Class c1 = person.getClass();
        System.out.println(c1.hashCode());

        // 方法二：通过forname 获得
        Class c2 = Class.forName("com.hui.pojo.Stuent");
        System.out.println(c2.hashCode());

        // 方法三：通过类名.class 获得 ，该方法最为可靠，程序性能最高
        Class c3 = Stuent.class;
        System.out.println();
        System.out.println(c3.hashCode());

        // 方法四：基本内置包装类有个Type 属性
        Class c4 = Integer.TYPE;

        // 获得父类类型
        Class c5 = c1.getSuperclass();
        // 打印：student的父类 com.hui.pojo.Person
        System.out.println(c5);
    }
}

class Person {
    String name;

    public Person() {
    }

    public Person(String name) {
        this.name = name;
    }

    @Override
    public String toString() {
        return "Person{" +
                "name='" + name + '\'' +
                '}';
    }
}

class Stuent extends Person {
    public void Student() {
        this.name = "学生";
    }
}

class Teacher extends Person {
    public void Teach() {
        this.name = "老师";
    }
}
// 已知某个类的实例，通过getClass()方法获取Class对象
Person person = new Person();
Class c1 = person.getClass();

// 已知一个类的全类名，且该类在类路径下，通过Class静态方法forName() 获取
Class c2 = Class.forName("com.hui.pojo.Stuent");

// 已知具体的类，通过.class 实例获得： 安全可靠，程序性能最高
Class c3 = Stuent.class;

内存分析

Java内存分为堆、栈、方法区

  • 堆：存放new对象和数组

    • 可被所有线程共享，不会存放别的对象引用

  • 栈：存放基本变量类型（包含具体数值）

    • 引用对象的变量（存放引用在堆里面的具体地址）

  • 方法区：可以被所有线程共享

  • 包含了所有的class和static变量

类的加载

  • 加载：将class内容加载进内存

    • 将静态数据转换为方法区运行时的结构数据
    • 生成一个代表此类的java.lang.Class对象（通过反射）

  • 链接：将java的二进制合并到JVM的运行环境中的一个过程

    • 验证：确保是否符合规范
    • 准备：正式为变量（static）分配内存，并设置默认初始值阶段，内存在方法区进行分配
    • 解析：虚拟机常量池符号引用（常量名）替换为直接引用（地址）的过程

  • 初始化：执行构造器()方法的过程

package com.hui.pojo;

public class Test05 {
    public static void main(String[] args) {
        A a = new A();
        System.out.println(A.m);

        /*
        1. 加载进内存，产生对应Class对象
        2. 链接，链接结束后 m = 0
        3. 初始化，执行<clinit>()构造方法，将所有静态方法合并
            <clinit>() {
                System.out.println("A类static初始化");
                m = 300;
                m = 100;
            }
        最后m = 100
         */
    }
}

class A{
    static {
        System.out.println("A类static初始化");
        m = 300;
    }

    static int m = 100;
    public A() {
        System.out.println("A类无参构造");
    }
}

/* 结尾输出
A类static初始化
A类无参构造
100
*/

类的引用

引用会分为主动引用和被动引用：

  • 主动引用会发生类的初始化，被动引用不会发生类的初始化

  • 主动引用产生：

    • 虚拟机启动，初始化main所在的类
    • new 一个新的对象
    • 调用静态成员和静态方法（除final）
    • 通过反射对类进行调用(java.lang.reflect)
    • 初始类时：若父类未被初始化，则初始化它的父类(类的执行过程)
public class Test06 {
    static {
        System.out.println("Main类加载！");
    }

    public static void main(String[] args) {
        // new 加载对象 主动
        Son son = new Son();
        // 反射 加载对象 主动
        Class c1 = Class.forName("com.hui.pojo.Son");
    }
}
class Father {
    static {
        System.out.println("加载父类！");
    }
}
class Son extends Father{
    static {
        System.out.println("加载子类！");
    }
}

/*
Main类加载！
加载父类！
加载子类！
*/

  • 类的被动引用：

    • 访问静态域时，真正声明此域的累才会被初始化：当通过子类引用父类静态变量，不会导致子类初始化
    • 通过数组定义类引用，不会触发此类初始化
    • 引用常量不会触发此类初始化（常量在链接时就存入调用类的常量池中了）
package com.hui.pojo;

public class Test06 {
    static {
        System.out.println("Main类加载！");
    }
    public static void main(String[] args) {
        /* 
        	子类调用父类静态方法，不会加载子类
           输出：Main类被加载！ -> 父类被加载 -> 2
         */
        System.out.println(Son.b);
        /*
            空间命名不会加载类
            输出：Main类被加载！
         */
        Son[] array = new Son[5];
        /*
            常量在链接阶段就存入调用类的常用池中
            输出：Main类被加载！ -> 1
         */
        System.out.println(Son.M);
    }
}
class Father {
    static int b = 2;
    static {
        System.out.println("加载父类！");
    }
}
class Son extends Father{
    static {
        System.out.println("加载子类！");
    }
    static final int M = 1;
}

类加载器

  • 类加载器：将静态数据转换为方法区的运行时结构数据，在堆中生成一个代表这个类的java.lang.Class对象
  • 类缓存：维持加载缓存。缓存是为了提高效率

类加载器作用：把类装载进内存

  • 引导加载器：C++ 编写。负责JAVA平台，JVM自带加载器，负责加载Java核心库（如rt.jar），就是根目录下的文件。无法直接获取该加载器
  • 扩展类加载器：负责/jre/lib/ext目录下的jar包，加载进工作库
  • 系统类加载器：（APP 或 System 加载器）负责java -classpath项目内家暴加载进内存
package com.hui.pojo;

public class Test07 {
    public static void main(String[] args) {
        // 获取系统类的加载器
        // 输出：sun.misc.Launcher$AppClassLoader@18b4aac2
        ClassLoader systemClassLoader = ClassLoader.getSystemClassLoader();
        System.out.println(systemClassLoader);

        // 获得系统类加载器的父类 --> 扩展类加载器
        // 输出：sun.misc.Launcher$ExtClassLoader@511d50c0
        ClassLoader parent = systemClassLoader.getParent();
        System.out.println(parent);

        // 获得扩展类加载器的父类加载器 --> 根加载器
        // 输出：null
        ClassLoader parent1 = parent.getParent();
        System.out.println(parent1);
    }
}

双亲委派机制

假设定义一个java.lang.String，在加载时会一层层找，通过用户加载器 -> 扩展加载器 -> 根加载器

加载时发现此包属于根加载器 则自己定义的就没用了，检测安全性

获取类运行时的结构（了解）

获得类的名字

package com.hui.demo;

public class Test01 {
    public static void main(String[] args) throws ClassNotFoundException {
        Class c1 = Class.forName("com.hui.demo.User");

        // 包名 + 类名：com.hui.demo.User
        System.out.println(c1.getName());
        // 获得类名： User
        System.out.println(c1.getSimpleName());
    }
}

获得类的属性

反射的方法名 方法作用 其他
getFields() 获得类公有属性 返回数组
getDeclaredFields() 获得所有属性 返回数组
getField(name) 获取指定属性的值(公有属性) 传入公有属性名
getDeclaredField(name) 获取指定属性的值 传入属性名 
public static void main(String[] args) throws ClassNotFoundException, NoSuchFieldException {
    Class c1 = Class.forName("com.hui.demo.User");

    // 获得公有属性 getFields()
    // 截取返回：public java.lang.String com.hui.demo.User.str
    Field[] fields = c1.getFields();
    for (Field field : fields) {
        System.out.println(field);
    }
    /*
         获得所有属性 getDeclaredFields()
         截取返回：  private java.lang.String com.hui.demo.User.name/id/age
                public java.lang.String com.hui.demo.User.str
    */
    fields = c1.getDeclaredFields();
    for (Field field : fields) {
        System.out.println(field);
    }

    // 获取指定属性的值(公有属性)
    // 截取返回：public java.lang.String com.hui.demo.User.str
    Field name = c1.getField("str");
    System.out.println(name);

    // 获取指定属性的值
    // 截取返回：private java.lang.String com.hui.demo.User.name
    name = c1.getDeclaredField("name");
    System.out.println(name);
}

获得类的方法

反射的方法名 方法作用 其他
getMethods() 获得本类和父类全部public方法 返回数组
getDeclaredMethods() 获取本类的所有方法（包括私有） 返回数组
getMethod(name,type) 获得指定方法 传入类名和参数类型 
package com.hui.demo;

import java.lang.reflect.Method;

public class Test01 {
    public static void main(String[] args) throws ClassNotFoundException, NoSuchFieldException, NoSuchMethodException {
        Class c1 = Class.forName("com.hui.demo.User");


        // 获得本类和父类全部public方法
        /* 截取输出：method：public final native void java.lang.Object.notify()
        *        method：public void com.hui.demo.User.setId(int)
method：public final void java.lang.Object.wait(long,int) throws java.lang.InterruptedException
        * method：public native int java.lang.Object.hashCode()
        * ...
        * */
        Method[] methods = c1.getMethods();
        for (Method method : methods) {
            System.out.println("method：" + method);
        }

        // 获取本类的所有方法（包括私有）
        /* 截取输出：
             methods!!：public void com.hui.demo.User.setName(java.lang.String)
             methods!!：public int com.hui.demo.User.getId()
        */

        methods = c1.getDeclaredMethods();
        for (Method method : methods) {
            System.out.println("methods!!：" + method);
        }

        // 获得指定方法
        // 第二个参数传入该参数的参数类型
        // java中有重载：获取方法值时，需要值的类型作为参数
        /*返回值：
            public void com.hui.demo.User.setName(java.lang.String)
            public java.lang.String com.hui.demo.User.getName()
        */
        Method getName = c1.getMethod("getName", null);
        Method setName = c1.getMethod("setName", String.class);
        System.out.println(setName);
        System.out.println(getName);  
    }
}

获得类中构造器

反射的方法名 方法作用 其他
getConstructors() 获得public构造器 返回数组
getDeclaredConstructors() 获得public构造器 返回数组
getDeclaredConstructor(name,type) 获得指定构造器 传入构造器名和参数类型 
package com.hui.demo;

import java.lang.reflect.Constructor;
import java.lang.reflect.Method;

public class Test01 {
    public static void main(String[] args) throws ClassNotFoundException, NoSuchMethodException {
        Class c1 = Class.forName("com.hui.demo.User");


        // 获得public构造器
        /*输出：
            public com.hui.demo.User()
            public com.hui.demo.User(java.lang.String,int,int)Constructor
        */
        Constructor[] constructors = c1.getConstructors();
        for (Constructor constructor : constructors) {
            System.out.println(constructor);
        }

        // 获得所有构造器
        /*输出：
            public com.hui.demo.User()
            public com.hui.demo.User(java.lang.String,int,int)Constructor
        */
        Constructor[] constructor = c1.getDeclaredConstructors();
        for (Constructor constructor1 : constructor) {
            System.out.println(constructor1);
        }

        // 获得指定构造器：public com.hui.demo.User(java.lang.String,int,int)
        Constructor declaredConstructor = c1.getDeclaredConstructor(String.class, int.class, int.class);
        System.out.println(declaredConstructor);
    }
}
